 Story — "The Royal Vault"

A kingdom has one treasury. No matter which minister requests it, they must all get the same vault — there cannot be two.

public class RoyalVault {
    private static volatile RoyalVault instance;
    private int goldCoins = 10_000;

    private RoyalVault() {}   // no public constructor

    public static RoyalVault getInstance() {
        if (instance == null) {
            synchronized (RoyalVault.class) {
                if (instance == null)
                    instance = new RoyalVault();
            }
        }
        return instance;
    }

    public void deposit(int coins) { goldCoins += coins; }
    public int  balance()          { return goldCoins; }
}

// --- Main.java ---
RoyalVault ministerA = RoyalVault.getInstance();
RoyalVault ministerB = RoyalVault.getInstance();
ministerA.deposit(500);
System.out.println(ministerB.balance());
System.out.println(ministerA == ministerB);
Input
ministerA.deposit(500)
Output

 Story — "The Weapon Forge"

A blacksmith's forge creates weapons. Elven forges craft bows; Dwarven forges craft axes. Client code orders a weapon without knowing how it's made.

interface Weapon { String describe(); }

class Bow  implements Weapon { public String describe(){ return "Elven Bow";   } }
class Axe  implements Weapon { public String describe(){ return "Dwarven Axe"; } }

abstract class Forge {
    abstract Weapon craft();          // factory method
    void deliver() {
        System.out.println("Delivering: " + craft().describe());
    }
}
class ElvenForge  extends Forge  { Weapon craft(){ return new Bow(); } }
class DwarvenForge extends Forge  { Weapon craft(){ return new Axe(); } }

// --- Main.java ---
Forge f1 = new ElvenForge();   f1.deliver();
Forge f2 = new DwarvenForge(); f2.deliver();
Input
ElvenForge → deliver()
DwarvenForge → deliver()
Output
Delivering: Elven Bow
Delivering: Dwarven Axe

 Story — "The Tavern Meal Order"

A customer orders a meal at a tavern step by step. The chef builds it only when the customer says "done". Another customer clones last night's special.

class Meal {
    String main, side, drink;
    public String toString(){
        return main + " + " + side + " + " + drink;
    }
}
class MealBuilder {
    private final Meal meal = new Meal();
    public MealBuilder main(String m)  { meal.main  = m; return this; }
    public MealBuilder side(String s)  { meal.side  = s; return this; }
    public MealBuilder drink(String d) { meal.drink = d; return this; }
    public Meal build() { return meal; }
}

Meal order = new MealBuilder()
    .main("Roast Chicken")
    .side("Bread")
    .drink("Ale")
    .build();
System.out.println(order);
Input
main=Roast Chicken
side=Bread, drink=Ale
Output
Roast Chicken + Bread + Ale

 Story — "The Magic Translator"

A modern wizard speaks English, but the ancient spell book is in Latin. An Adapter translates on the fly so the wizard can cast spells without learning Latin.

interface Spell       { void cast(); }
class LatinSpell      { void incantare(){ System.out.println("Ignis!"); } }

class SpellAdapter implements Spell {
    private final LatinSpell latin;
    SpellAdapter(LatinSpell l) { this.latin = l; }

    public void cast() {
        System.out.print("[Adapter] Translating → ");
        latin.incantare();      // delegates to legacy
    }
}

// Wizard only knows Spell interface
Spell fireball = new SpellAdapter(new LatinSpell());
fireball.cast();
Input

Output
[Adapter] Translating → Ignis!

 Story — "The Army Roster"

A general commands units: individual soldiers and squads. Either way, the order "march!" propagates uniformly down the tree.

interface Unit { void march(); }

class Soldier implements Unit {
    String name;
    Soldier(String n){ name = n; }
    public void march(){ System.out.println(name + " marches."); }
}
class Squad implements Unit {
    private List<Unit> members = new ArrayList<>();
    void add(Unit u){ members.add(u); }
    public void march(){ members.forEach(Unit::march); }
}

Squad alpha = new Squad();
alpha.add(new Soldier("Aria"));
alpha.add(new Soldier("Bran"));

Squad army = new Squad();
army.add(alpha);
army.add(new Soldier("Cael"));
army.march();
Input
army.march()
Output
Aria marches.
Bran marches.
Cael marches.

 Story — "The Enchanted Sword"

A plain sword can be enchanted with fire, then ice — each enchantment wraps the previous, stacking damage bonuses without changing the sword itself.

interface Sword    { int damage(); }

class IronSword implements Sword {
    public int damage(){ return 10; }
}
abstract class Enchantment implements Sword {
    protected Sword wrapped;
    Enchantment(Sword s){ wrapped = s; }
}
class FireEnchant extends Enchantment {
    FireEnchant(Sword s){ super(s); }
    public int damage(){ return wrapped.damage() + 5; }
}
class IceEnchant extends Enchantment {
    IceEnchant(Sword s){ super(s); }
    public int damage(){ return wrapped.damage() + 3; }
}

Sword s = new IceEnchant(new FireEnchant(new IronSword()));
System.out.println("Total damage: " + s.damage());
Input
IronSword + Fire + Ice
Output
Total damage: 18

 Story — "The Travelling Merchant"

A merchant chooses how to sort his inventory based on market day: by price, by weight, or by name — the sorting strategy swaps without changing the cart.

interface SortStrategy {
    void sort(List<String> items);
}
class AlphaSort implements SortStrategy {
    public void sort(List<String> items){
        Collections.sort(items);
    }
}
class LengthSort implements SortStrategy {
    public void sort(List<String> items){
        items.sort(Comparator.comparingInt(String::length));
    }
}
class MerchantCart {
    private SortStrategy strategy;
    void setStrategy(SortStrategy s){ strategy = s; }
    void display(List<String> items){
        strategy.sort(items);
        System.out.println(items);
    }
}
List<String> goods = new ArrayList<>(List.of("Silk","Iron","Gem"));
MerchantCart cart = new MerchantCart();
cart.setStrategy(new AlphaSort());  cart.display(goods);
cart.setStrategy(new LengthSort()); cart.display(goods);
Input
[Silk, Iron, Gem]
Strategy 1: Alpha
Strategy 2: Length
Output
[Gem, Iron, Silk]
[Gem, Silk, Iron]

 Story — "The Town Crier"

When the king makes an announcement, the town crier automatically notifies all registered citizens. No citizen needs to poll the palace.

interface Citizen      { void hear(String news); }

class Palace {
    private List<Citizen> citizens = new ArrayList<>();
    void register(Citizen c){ citizens.add(c); }
    void announce(String msg){
        citizens.forEach(c -> c.hear(msg));
    }
}

Palace palace = new Palace();
palace.register(news -> System.out.println("Alice hears: " + news));
palace.register(news -> System.out.println("Bob hears: "   + news));
palace.announce("Festival tomorrow!");
Input
announce("Festival tomorrow!")
Output
Alice hears: Festival tomorrow!
Bob hears: Festival tomorrow!

 Story — "The Hero's Quest"

Every quest follows the same skeleton: prepare → fight → claim reward. But elves prepare differently from warriors — the steps vary, not the skeleton.

abstract class Quest {
    final void undertake() {   // template method
        prepare();
        fight();
        claimReward();
    }
    abstract void prepare();
    void fight()  { System.out.println("Fighting the dragon!"); }
    void claimReward(){ System.out.println("Reward claimed."); }
}
class ElfQuest extends Quest {
    void prepare(){ System.out.println("Elf: sharpens arrows."); }
}
class WarriorQuest extends Quest {
    void prepare(){ System.out.println("Warrior: polishes armour."); }
}

new ElfQuest().undertake();
System.out.println("---");
new WarriorQuest().undertake();

 Story — "The Castle Gate Guards"

A visitor approaches the castle. First the page checks rank, then the knight checks allegiance, then the king makes the final call — each guard passes up if they can't decide.

abstract class Guard {
    protected Guard next;
    Guard setNext(Guard g){ next = g; return g; }
    abstract void handle(int rank);
}
class Page extends Guard {
    void handle(int r){
        if(r < 2) System.out.println("Page lets in commoner.");
        else if(next != null) next.handle(r);
    }
}
class Knight extends Guard {
    void handle(int r){
        if(r < 5) System.out.println("Knight admits noble.");
        else if(next != null) next.handle(r);
    }
}
class King extends Guard {
    void handle(int r){ System.out.println("King receives dignitary."); }
}

Guard gate = new Page();
gate.setNext(new Knight()).setNext(new King());
gate.handle(1);
gate.handle(4);
gate.handle(9);
Input
rank=1
rank=4
rank=9
Output
Page lets in commoner.
Knight admits noble.
King receives dignitary.

 Story — "The Overstretched Innkeeper"

The Innkeeper does everything: cooks, cleans, charges, repairs — a God Object. We refactor by giving each responsibility to a dedicated class.

// ❌ BEFORE — God Object
class Innkeeper {
    void cook()   { ... }
    void clean()  { ... }
    void charge() { ... }
    void repair() { ... }
}

// ✅ AFTER — Single Responsibility
class Cook    { void cook()   { System.out.println("Meal ready.");   } }
class Cleaner { void clean()  { System.out.println("Room clean.");   } }
class Cashier { void charge() { System.out.println("Bill settled."); } }

new Cook().cook();
new Cleaner().clean();
new Cashier().charge();
Input
Each class called once
Output
Meal ready.
Room clean.
Bill settled.
Week 12 · Capstone

Full System Design Project

Apply multiple patterns together to design and implement a cohesive mini-application from scratch.

Project — "Realm Manager"

Design a turn-based kingdom simulation applying at minimum: Singleton (GameEngine), Factory (Unit creation), Observer (event log), Strategy (combat), Command (undo moves), Composite (army tree), Facade (GameAPI).

Deliverables
  • UML class diagram annotated with pattern names
  • Working Java implementation (≥ 5 patterns)
  • Justification doc: why each pattern was chosen
  • 10-minute demo presentation
Assessment criteria
  • Correct pattern intent and structure (40%)
  • Code quality: SOLID, clean naming, no anti-patterns (30%)
  • Interaction between patterns (20%)
  • Presentation clarity (10%)
 Sample skeleton
// Singleton + Facade together
class GameEngine {                      // Singleton
    private static GameEngine instance;
    private EventLog  log  = new EventLog();   // Observer hub
    private UnitFactory factory = new UnitFactory(); // Factory

    public static GameEngine get(){
        if(instance==null) instance = new GameEngine();
        return instance;
    }
    public Unit recruit(String type){
        Unit u = factory.create(type);
        log.broadcast(type + " recruited.");
        return u;
    }
}
// Main
Unit archer = GameEngine.get().recruit("Archer");
Unit mage   = GameEngine.get().recruit("Mage");
Input
recruit("Archer")
recruit("Mage")
Output
Archer recruited.
Mage recruited.
